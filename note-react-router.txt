client atau server side
- saat kita membuat web, terdapat 2 pilihan yang bisa kita pilih, CSR (client side render), atau SSR (server side render)
- saat kita menggunakan frontend framework seperti react, biasanya pilihan pertama adalah menggunakan CSR (client side render)
- hal ini menjadikan bagian server tidak menangani halaman web lagi dan tidak perlu membuat HTML lagi, semua dikerjakan oleh (device/browser) client oleh react
- secara otomatis CSR akan semakin cepat dibandingkan SSR, karena komunikasi ke server bisa dibuat minimal dan data yang dikirim/diterima tidak terlalu besar

server side render
1. user melakukan request (url) ke server (app)
2. di sisi server akan memanipulasi dan mengkalkulasi data yang diperlukan dan mengembalikan ke browser (user) berupa HTML
3. disisi browser (milik user) akan melakukan render HTML sesuai dengan informasi yang sudah dilakukan

client side render
1. user melakukan request cukup melalui browser
2. request akan ditangani oleh react
3. react, menggunakan ajax untuk berkomunikasi ke server melalui API
4. kemudian server akan mengembalikan response API dalam bentuk json yang dikirimkan ke react (browser)
5. dan react akan menampilkan nya ke user

MPA (multi page application)
- sebelumnya, ketika belajar react dasar, kita membuat banyak sekali halaman HTML, dan tiap halaman HTML menampilkan component react
- hal ini artinya walaupun aplikasi yang sudah kita buat adalah CSR, namun tetap multi page 
- problem dengan MPA (multi page application) adalah, setiap berganti halaman, maka browser akan menjalankan ulang program react-nya dari awal lagi (reload/render ulang dari awal)
- hal ini menyebabkan walaupun sudah menggunakan CSR, namun kemungkinan tidak optimal karena harus di load ulang semuanya ketika berpindah halaman

SPA (Single Page Application)
- kebalikan dari MPA, adalah SPA, dimana kita hanya membuat satu halaman HTML
- nanti halaman itu yang akan menerima semua request, dari isi halamannya akan berubah sesuai dengan request yang diminta oleh User atau sesuai response dari server
- keuntungan dari SPA adalah perpindahan halaman lebih cepat, karena hanya mengubah isi content saja, tidak harus me-load ulang seluruh halaman lagi
- namun sayangnya, secara default, react tidak mendukung SPA dengan baik
- kita harus lakukan semuanya secara manual jika ingin mengganti-ganti isi dari halaman, dan itu sangat rumit dilakukan

router library
- salah satu hal yang bisa kita lakukan dibanding mengganti-ganti halaman secara manual, adalah menggunakan library khusus untuk itu, yaitu router library
- router library adalah library yang digunakan agar component yang ditampilkan bisa berbeda-beda tergantung dari URL yang diakses
- dengan begitu, kita bisa fokus membuat component untuk halaman, tidak perlu memikirkan lagi kompleksitas bagaimana cara melakukan routing (menentukan component mana yang ditampilkan untuk URL yang diakses)
- untungnya, banyak sekali router library yang tersedia untuk react

react router
- ada banyak sekali router library yang bisa kita gunakan untuk react
- referensi : https://react.libhunt.com/libs/router/react
- namun pada materi ini, kita akan menggunakan salah satu router library yang populer, yaitu adalah React Router
- referensi : https://reactrouter.com/
- react router adalah salah satu library untuk routing yang opensource dan saat ini sangat populer digunakan di ekosistem react
- referensi : https://github.com/remix-run/react-router

membuat project
- npm create vite@latest belajar-reactjs-router -- --template react
- upgrade versi react ke versi 19

menambahkan library react router
- npm install react-router

setup react router
- berbeda saat dengan MPA, ketika membuat SPA, kita cukup membuat satu halaman html sebagain entry point (titik masuk) semua request
- nanti, penentuan komponen mana yang akan ditampilkan, akan diurus oleh react router

browser router
- ketika menggunakan react router, saat membuat/melakukan render komponen menggunakan react, kita bisa bungkus aplikasi react kita menggunakan browser router
- referensi : https://api.reactrouter.com/v7/functions/react-router.BrowserRouter.html

routing
- routing adalah pemetaan antara URL path dan komponen yang akan ditampilkan
- untuk melakukan routing, kita bisa menggunakan komponen <Routes>
- referensi : https://api.reactrouter.com/v7/functions/react-router.Routes.html
- dan untuk pemetaan tiap routing nya, kita bisa gunakan komponen <Route>
- referensi : https://api.reactrouter.com/v7/functions/react-router.Route.html

nested route
- saat nanti kita membuat routing, kadang terdapat routing yang memiliki prefix yang sama
- jika kita harus buat satu per-satu, maka akan tidak efektif ketika pembuatan kode nya
- untungnya, react router mendukung nested route
- kita bisa membuat komponen route di dalam komponen route, dan secara otomatis Route path diatas nya akan digunakan sebagai prefix untuk child Route nya

index route
- komponen route memiliki attribute spesial bernama index
- attribute ini digunakan sebagai default halaman index
- ini biasanya digunakan untuk path awal, misal '/', atau untuk nested path '/data/...'
- contoh sebelumnya, saat kita membuka '/data/', maka tidak ada komponen yang ditampilkan, kita bisa tampilkan komponen yang kita mau dengan menggunakan route index
- misalnya, sekarang kita akan buat komponen data untuk route index

outlet
- saat kita membuat halaman web, kadang beberapa halaman memiliki layout yang sama. misal header dan footer nya sama, namun isi content nya berbeda
- sebenarnya kita bisa menggunakan JSX untuk melakukan ini dengan menggunakan children
- namun, react router memiliki cara yang lebih mudah, kita bisa membuat route dengan component yang digunakan sebagai layout
- lalu kita bisa tambahkan route didalamnya yang digunakan sebagai children
- namun kita tidak menggunakan children props lagi, melainkan menggunakan component Outlet
- referensi : https://api.reactrouter.com/v7/functions/react-router.Outlet.html

route param
- saat kita membuat URL path, kadang kita tidak membuat path yang statis, kadangkala butuh membuat path yang dinamis
- misalnya kita ingin menambahkan 'id' / kode pada path, misal '/product/123', dimana '123' adalah id dari data Product
- di react router, kita juga bisa lakukan hal ini, dengan menambahkan tanda ':' (titik dua), diikuti dengan nama parameter(param). Contoh: '/products/:id', id dianggap sebagai parameter dinamis
- jika kita menggunakan route '/product/123', maka '123' dianggap sebagai data parameter id

useParams()
- saat kita membuat path yang dinamis, biasanya kita ingin mendapatkan informasi dari path yang dinamis tersebut
- seluruh dynamic param yang terdapat di URL path secara otomatis akan disimpan dalam object, dan kita bisa mengambilnya menggunakan function 'useParams()'
- referensi : https://api.reactrouter.com/v7/functions/react-router.useParams.html
- misal, sekarang kita akan coba tambahankan useParams() pada komponen ProductDetail

mutiple route param
- kita bisa menambahkan dynamic param pada path lebih dari satu
- yang penting kita gunakan nama dynamic param yang berbeda, misal: '/users/:userId/addresses/:addressId
- maka kita bisa dapatkan informasi userId dan addressId di object hasil dari 'useParams()'

star segment
- jika route path diakhiri dengan '/*' (star segment), maka itu akan cocok dengan karakter apapun, termasuk karakter slash '/' itu sendiri
- misal, ketika kita memiliki path '/files/*', maka akan cocok dengan '/files/contoh' dan '/files/lagi/contoh'
- star segment juga bisa diambil nilainya pada 'useParams()'

star segment sebagai Not Found page
- apa yang terjadi jika kita mengakses URL path yang tidak tersedia di router?
- kita akan mendapatkan error bahwa tidak ada route yang cocok dengan path yang kita akses
- salah satu yang biasa dilakukan di react router adalah, kita bisa menambahkan star segment untuk path '/*' pada bagian bawah sebagai handler untuk menangani halaman Not Found
- sekarang kita akan coba buat komponen NotFound, dan registrasikan ke route path '/*'

navigation
- salah satu yang biasa kita lakukan saat membuat web, adalah berpindah dari satu halaman ke halaman yang lainnya
- biasanya untuk perpindahan halaman, kita biasanya akan menggunakan anchor element, yaitu menggunakan tag '<a>'
- namun permasalahannya adalah, ketika mengunakan anchor element, maka browser akan melakukan reload ke halaman baru, yang artinya halaman akan dimuat ulang, termasuk object react juga akan dibuat ulang semuanya
- hal ini mungkin akan memperlambat proses perpindahan halaman

link element
- react router menyediakan komponen link
- komponen link ini digunakan sebagai pengganti anchor element
- berbeda dengan anchor element biasanya, komponen link ini tidak akan melakukan reload halaman
- referensi : https://api.reactrouter.com/v7/functions/react-router.Links.html

link to
- attribute to di komponen Link bisa menerima parameter object, dimana terdapat attribute 'pathname', 'search' dan 'hash'
- attribute pathname untuk lokasi path
- attribute search untuk query parameter
- dan attribute hash untuk #fragmen

nav link
- salah satu yang biasa kita lakukan ketika membuat link adalah, membedakan style link yang sedang aktif dan yang tidak aktif
- biasanya, kita lakukan pengecekan, jika URl path yang sedang dikunjungi saat ini sama dengan tujuan link, maka kita ubah style nya menjadi aktif
- react router menyediakan komponen khusus untuk mempermudah hal ini, cara penggunaanya sama seperti komponen link, namun bedanya jika URL path saat ini sama dengan tujuan dari navlink, maka secara otomatis NavLink akan menggunakan style 'a.active' (memiliki nilai aktif atau tidak)
- referensi : https://api.reactrouter.com/v7/variables/react-router.NavLink.html

use navigate
- pada kasus tertentu, mungkin kita ingin melakukan navigasi dari satu halaman ke halaman lain menggunakan javascript
- untuk melakukan ini, react router menyediakan hooks 'useNavigate()'
- referensi : https://api.reactrouter.com/v7/functions/react-router.useNavigate.html
- useNavigate() akan mengembalikan function yang bisa kita gunakan untuk berpindah ke halaman lain
- referensi : https://api.reactrouter.com/v7/interfaces/react-router.NavigateFunction.html
- kita bisa gunakan parameter path
- referensi : https://api.reactrouter.com/v7/interfaces/react-router.Path.html
- atau number (untuk maju/mundur), contoh: -1 (untuk mundur satu kali), -2 (untuk mundur ke halaman sebelumnya dua kali), dst

use search param
- saat menggunakan URL, kadang kita akan memafaatkan query parameter untuk mengirim data
- react router menyediakan hook 'useSearchParams()' yang bisa kita gunakan untuk mendapatkan data query parameter
- referensi : https://api.reactrouter.com/v7/functions/react-router.useSearchParams.html
- useSearchParams() akan mengembalikan array yang berisi object URLSearchParams dan setter function nya
- referensi : https://developer.mozilla.org/en-US/docs/Web/API/URLSearchParams

use location
- untuk mendapatkan URL saat ini, sebenarnya kita bisa menggunakan 'window.location.href'
- tapi react router menyediakan hook 'useLocation()', yang bisa digunakan untuk mendapatkan lokasi path saat ini
- referensi : https://api.reactrouter.com/v7/functions/react-router.useLocation.html
- hasil dari return dari 'useLocation()' adalah object location, sehingga kita bisa dapat informasi lengkap dari location, seperti path, query sampai ke fragment #fragment
- referensi : https://api.reactrouter.com/v7/interfaces/react-router.Location.html
